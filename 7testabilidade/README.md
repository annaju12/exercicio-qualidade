## Problema no código ruim
O código cria dependências diretamente dentro da classe, dificultando a criação de testes automatizados.

## Por que o código bom é melhor
O código bom recebe a dependência por injeção, permitindo o uso de mocks em testes.

## O que foi aplicado para melhorar
- Injeção de dependência
- Desacoplamento
- Facilidade para testes automatizados
