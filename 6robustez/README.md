## Problema no código ruim
O código não trata situações de erro, como a divisão por zero, o que pode causar falhas na execução do programa.

## Por que o código bom é melhor
O código bom valida a entrada antes de realizar a operação, evitando erros inesperados.

## O que foi aplicado para melhorar
- Validação de dados
- Tratamento de exceções
- Maior confiabilidade do sistema
