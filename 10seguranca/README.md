## Problema no código ruim
O código está vulnerável a ataques de SQL Injection ao concatenar diretamente dados do usuário.

## Por que o código bom é melhor
O código bom utiliza Prepared Statements, protegendo o sistema contra injeção de SQL.

## O que foi aplicado para melhorar
- Segurança da aplicação
- Validação de entradas
- Boas práticas de acesso a banco de dados
